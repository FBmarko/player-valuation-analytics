# UI Data Requirement Map

Future source classes:

- `DATASET_DIRECT`: can come directly from processed CSV columns.
- `MODEL_OUTPUT`: comes from trained model prediction.
- `DERIVED_SCORE`: calculated from multiple stats or normalized dataset columns.
- `TEMPLATE_GENERATED`: generated by rule-based text templates.
- `FRONTEND_CONFIG`: pure UI/display configuration.
- `MISSING_OR_UNKNOWN`: needed by UI but no clear source was found.

## Dashboard / Command Center

Required fields:

- `players.length`
- `teams[*].league`
- `players[*].aiQualityScore`
- `players[*].marketValueHistory[-1].value`

Current source:

- `players` and `teams` from `src/data/mockData.js`
- Derived locally in `src/pages/Home.jsx`

Future source:

- `players.length`: `DATASET_DIRECT` after filtering eligible frontend player records
- `teams[*].league`: `DATASET_DIRECT`
- `players[*].aiQualityScore`: `DERIVED_SCORE` or `MODEL_OUTPUT`, depending on final scoring design
- latest market value: `DATASET_DIRECT` if using `PD_Guncel`; `MODEL_OUTPUT` if using predicted value

Notes:

- The current "Portfolio market value" sums fake latest market history values.
- Decide whether dashboard value should mean current Transfermarkt value (`PD_Guncel`) or predicted model value.
